# Community Center Cooling Optimization
헬스장 출입기록 기반 냉방 운영 최적화 프로젝트

출입(수요) 패턴으로 혼잡 시간대를 정의하고,
냉방 운영을 ‘민원 즉시 대응’에서 ‘표준 룰’로 전환하기 위한 기준안을 만들었습니다.

---

## ✅ 배경 (Background)
여름철(6–7월) 냉방 관련 민원이 반복적으로 발생했습니다.

“덥다 / 춥다” 요청에 맞춰 커뮤니티센터 직원이 직접 수동 조정을 했기 때문에  
근무자나 시간대에 따라 대응 방식이 달라졌습니다.

관리사무소에서는 냉방비 절약을 위해 온도를 고정하라고 권유했습니다.

​---

## 🎯 목표(Goal)
냉방 의사결정을 개인 판단이 아닌 **데이터 기반 운영 기준**으로 전환합니다.

이를 위해
​
- 피크/비피크/예외 상황을 정의
- 비피크 운영 기준 정의
- 예외 상황 정의

를 수행하고 **운영 기준 메뉴얼을 제작**합니다.

---

## 🧱 데이터 (Input Data)
### 내부 데이터
커뮤니티센터 출입 기록​
* 기간: 2025-06-01 ~ 2025-07-10​
* 형태: Google Sheet​

### 외부 데이터
해당기간 일 단위 기상(온도/습도)​
* 기간: 2025-06-01 ~ 2025-07-10​
* 출처: 기상청 API

---

## 📌 지표 설계 
1️⃣ 요일/시간대별로 혼잡(수요)이 언제 집중되는가? 
→ 피크 시간대 정의.

2️⃣ 혼잡이 낮은 시간대는 언제인가? 
→ 비피크 운영(절감) 기준 정의.

3️⃣ 더운 날/특정 조건에서 패턴이 달라지는가? 
→ 예외 조건(기상 등) 정의.​

---

## 📊 분석 (Method)
BigQuery SQL을 사용하여

- 시간대별 출입 건수 집계
- 요일 × 시간대 패턴 분석
- 피크 / 비피크 구간 정의

를 수행했습니다.

## ⚠️ Limitations

​
에 초점을 두었습니다.
