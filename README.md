# Community Center Cooling Optimization
헬스장 출입기록 기반 냉방 운영 최적화 프로젝트

출입(수요) 패턴으로 혼잡 시간대를 정의하고, 냉방 운영을 ‘민원 즉시 대응’에서 ‘표준 룰’로 전환하기 위한 기준안을 만들었습니다.

---

## ✅ 배경 (Background)
- 여름철(6–7월) 냉방 관련 민원이 반복적으로 발생했습니다.
  ​
- “덥다/춥다” 요청에 맞춰 커뮤니티센터 직원이 직접 수동 조정을 했기에 근무자·시간대에 따라 대응이 달라졌습니다.
  ​
- 관리사무소에서는 냉방비 절약을 위해 온도를 고정하라고 권유했습니다.
​

---

## 🎯 목표(Goal)
- 냉방 의사결정을 개인 판단이 아닌 합의된 룰로 운영 가능하게 만들어야합니다.
  ​
- 피크/비피크/예외 상황을 정의하고, 누구나 같은 기준으로 대응할 수 있게 문서화해야합니다.

- 산출물로 운영 기준안 메뉴얼 1개를 제작합니다.


---

## 🧱 데이터 (Intput Data)
내부 데이터: 출입 기록(커뮤니티센터)​
* 기간: 2025-06-01 ~ 2025-07-10​
* 형태: Google Sheet(정리 후 분석용 테이블로 사용)​

외부 데이터: 일 단위 기상(온도/습도)​
* 기간: 2025-06-01 ~ 2025-07-10​
* 출처: 기상청 API


---

## 📌 지표 설계 
Q1. 요일/시간대별로 혼잡(수요)이 언제 집중되는가? → 피크 시간대 정의.

Q2. 혼잡이 낮은 시간대는 언제인가? → 비피크 운영(절감) 기준 정의.
​
Q3. 더운 날/특정 조건에서 패턴이 달라지는가? → 예외 조건(기상 등) 정의.​


---

## 📊 분석 (Method)
- 시간대×요일별 출입 건수를 집계해 수요 패턴을 만들었습니다.
​
- 이 패턴을 바탕으로 피크/비피크 구간을 규칙으로 고정하고, 현장 필요 시 예외 조건을 보강했습니다.


---

## 🧾 산출물 (Deliverables)
- 운영 기준안: 피크/비피크/예외 규칙 요약(현장에서 바로 참고 가능한 형태)했습니다.
  ​
- 대시보드: 요일×시간대 히트맵으로 혼잡 구간을 한눈에 확인 할 수 있습니다.
          피크/비피크/예외 룰을 함께 보며 운영 액션을 선택합니다(냉방 강도/환기/설정값 등).​

  ​
- 핵심 SQL: 같은 입력이면 같은 룰이 나오도록 재현 가능한 쿼리로 정리했습니다.​
